# player 规则匹配示例清单

> 部署与日常运行见 [README.md](../README.md)。

## 播放控制类

- `开始播放`、`播放音乐`、`开始` -> `RULE_CMD_PLAY_START`
- `暂停`、`停一下` -> `RULE_CMD_PAUSE`
- `继续`、`继续播放` -> `RULE_CMD_RESUME`
- `下一首`、`换一首`、`下一曲` -> `RULE_CMD_NEXT`
- `上一首`、`上一曲` -> `RULE_CMD_PREV`
- `结束播放`、`停止`、`我不想听了` -> `RULE_CMD_STOP`

## 搜歌类（RULE_CMD_PLAY_QUERY）

- `我想听刘宇宁的歌` -> 提取 `刘宇宁`，按文本搜歌
- `来一首你从未离去` -> 提取 `你从未离去`，按文本搜歌
- `播放稻香` -> 提取 `稻香`，按文本搜歌
- `我想听白天的你从未离去` -> 提取 `白天的你从未离去`，按文本搜歌

## 泛听歌映射（热门伪随机）

- `我想听歌`、`想听歌`、`听歌` -> 映射为 `热门`
- `我想听音乐`、`想听音乐`、`播放音乐` -> 映射为 `热门`
- `我想听`、`想听`（无具体关键词） -> 映射为 `热门`
- `纯音乐`、`轻音乐` -> 不映射热门，按原词搜歌

## 非规则文本（RULE_CMD_NONE）

- `介绍一下自己`、`今天天气如何`、`你是联网的吗`、`王维有什么诗` -> 规则未命中，直接交给 LLM，再交给 TTS
- `RULE_CMD_NONE` 不再尝试 `try_music_lib_play`，避免把问答句误判为搜歌

## 播放模式（顺序/单曲）

- `顺序播放`、`列表播放` 等 -> `RULE_CMD_MODE_ORDER`，切换为顺序播放并播放预生成提示「已切换到顺序播放模式」
- `单曲循环`、`单曲重复` 等 -> `RULE_CMD_MODE_SINGLE`，切换为单曲循环并播放预生成提示「已切换到单曲循环模式」
- 提示 WAV 与唤醒词走同一套 TTS 内部播放路径，见 `docs/smart-speaker-client-播放模式与语音恢复策略更新说明.md`

## 在线/离线模式切换

- `离线模式` -> `RULE_CMD_SWITCH_OFFLINE`
- `在线模式` -> `RULE_CMD_SWITCH_ONLINE`
