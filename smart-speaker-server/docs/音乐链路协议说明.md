# 音乐链路协议说明

## 角色边界

- `music-service`：负责音乐能力查询与 URL 解析。
- `server`：负责 `client` / `qtapp` 转发、缓存最近一次状态与队列快照。
- `client`：维护真实播放队列、当前播放状态、切歌状态机。
- `qtapp`：展示真实队列并发送显式控制命令。

## 状态快照

`client -> server -> qtapp`

```json
{
  "cmd": "device_report",
  "deviceid": "0001",
  "cur_singer": "周杰伦",
  "cur_music": "七里香",
  "cur_mode": 0,
  "cur_volume": 60,
  "state": "play",
  "playlist_version": 12,
  "current_index": 3,
  "current_source": "wy",
  "current_song_id": "1901371647"
}
```

- `playlist_version`：队列结构变更版本。
- `current_index`：当前播放项在真实队列中的 0 基索引，未命中时为 `-1`。
- `current_source/current_song_id`：当前播放项稳定身份。

## 队列快照

`client -> server(cache) -> qtapp`

```json
{
  "cmd": "upload_music_list",
  "deviceid": "0001",
  "playlist_version": 12,
  "current_index": 3,
  "current_source": "wy",
  "current_song_id": "1901371647",
  "music": [
    {
      "title": "七里香",
      "subtitle": "周杰伦",
      "source": "wy",
      "id": "1901371647",
      "play_url": "https://..."
    }
  ]
}
```

- `music` 为真实完整队列，不再只传展示字符串。
- `server` 缓存最近一次 `upload_music_list`，兼容 `app_get_music_list` 时直接回缓存。
- `qtapp` 必须按 `source + id` 定位歌曲，不再按展示文案匹配。

## 切歌命令

`qtapp -> server -> client`

```json
{
  "cmd": "app_play_assign_song",
  "appid": "app001",
  "deviceid": "0001",
  "source": "wy",
  "id": "1901371647",
  "title": "七里香",
  "subtitle": "周杰伦",
  "music": "周杰伦/七里香"
}
```

- 新协议优先使用 `source/id/title/subtitle`。
- `music` 仅作为兼容旧端兜底字段。
- `client` 优先按 `source/id` 切歌，找不到时才退化到标题/歌手匹配。

## 音乐能力命令

- `music.search.song`
  - 精确搜歌；空关键词表示“默认榜单推荐页”。
- `music.search.playlist`
  - 搜歌单。
- `music.search.artist`
  - 搜歌手；当前实现优先支持 `wy`。
- `music.playlist.detail`
  - 拉取歌单歌曲列表。
- `music.artist.hot`
  - 拉取歌手热门歌；当前实现优先支持 `wy`。
- `music.leaderboard.list`
  - 拉取榜单列表。
- `music.leaderboard.detail`
  - 拉取榜单歌曲列表；`id/source` 为空时走服务端默认榜单配置。
- `music.url.resolve`
  - 解析 `source/id` 对应可播放 URL。

## 同步时机

- `client` 连上 `server` 后，`qtapp` 首次收到 `device_report` 可兼容触发一次 `app_get_music_list`。
- `client` 在整队列替换、单曲插入、在线翻页后主动推送最新 `upload_music_list`。
- `qtapp` 之后优先消费 `server` 转发或补发的 `upload_music_list`，不再依赖心跳重复拉列表。
- `qtapp` 重连后，`server` 会先补发缓存的最新 `device_report` 与 `upload_music_list`。
